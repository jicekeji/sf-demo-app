# 变更提案：个人待办事项应用（MyTODO）

## Why

当前仓库仅有一个静态骨架页面（`index.html`），尚无任何业务能力。项目奠基已批准，目标是构建一个「管理个人日常待办事项」的 TODO 应用。本变更为该 feature 建立首个规格：定义用户创建、组织、完成与持久化待办事项的核心行为，为后续实现（React + Vite 前端、Node HTTP API + PostgreSQL 后端）提供可验收的依据。

本阶段只产出规格，不写实现代码。

## What Changes

- 新增待办事项的增删改查能力：用户可创建、查看、编辑标题、删除待办事项。
- 新增完成状态能力：用户可将事项标记为已完成/未完成，并按状态筛选列表。
- 新增持久化能力：待办数据跨会话保存，数据迁移向后兼容（加列不删列、可重放）。
- 明确非功能约束：鉴权 fail-closed、结构化日志与健康探测遵循项目奠基基线。
- 复用已批准的设计系统与视觉主题（靛蓝主色 `#4f46e5`、设计令牌、组件清单）。

本阶段不改动 `index.html` 静态骨架，也不引入路由、协作/多用户共享、提醒推送等超出范围的特性。

## Capabilities

- `todo-management`：待办事项的创建、查看、编辑、删除。
- `todo-completion`：完成状态的标记切换与按状态筛选。
- `todo-persistence`：待办数据的跨会话持久化与向后兼容迁移。
